1. 마이페이지
  * 전체적인 특징 - 마이페이지이기 때문에 보안에 신경을 써 url을 통해 들어오는 get방식을 
		못들어오게 막아놓고, 로그인이 안되있으면 페이지에 들어올 수 없게
		하였다.
	     ex) request.getHeader("referer"); referer 값이 null이면 url을 통해 들어온 값

  * 메인 - 마이페이지 메인의 특징은 회원의 보유 포인트와 쿠폰 갯수가 실시간으로 반영되고,
	최근주문, 최근 문의내역 3건에 대한 항목을 보여준다.
	
  * 주문배송조회 -  각주문 상품에 대해 상품 상세페이지로 이동할 수 있게 링크를 걸어두었고,
		삭제된 상품에 대해서는 링크이동을 막아놓았다. 배송완료가 되었을 때
   		구매확정 버튼을 제공해 구매확정을 했을때 리뷰작성을 한번만 할 수 있게
		리뷰작성 버튼을 제공하였다.v

  * 관심상품 - 관심상품에서 장바구니로 이동할 수 있는 버튼을 제공하였고, 바로 구매할 수 있게
	      구매 버튼도 제공하였다.

  * 할인쿠폰 - 보유한 쿠폰에 대해 보여준다.(특별한 특징x)

  * 나의 리뷰 - 주문배송조회에서 작성한 리뷰내용을 페이징 처리하였고, 상세 내용은 각 항목에 대해 
	       숨겨놓고 아이콘을 제공해 아이콘 클릭시 볼 수 있게 하였다.

  * 나의 레시피 - 페이징 처리를 하였고 금메달, 은메달, 동메달에 선정된 레시피가 있으면 제일 먼저
	          정렬하게 만들었다.

  * 문의내역 - 페이징 처리를 하였고 답변이 달린경우 처리완료에 대해 시각적인 표현을 해주었다.

  * 개인정보 변경 - 보안을 위해 비밀번호를 치고 들어갈 수 있게 하였고, 유효성 검사를 통해 
		변경의 무분별함을 막아놓았다.

  * 배송지 관리 - 유효성 검사를 통해 빈값은 못넣게 막아놓았다.

  * 쪽지 관리 - 페이징 처리를 하였고 체크박스를 통해 선택적 삭제, 읽음 버튼을 제공해 
                     사용자의 편리함을 제공하였다.
	       검색 기능을 통해 쪽지 제목,내용을 통해 빠르게 찾아볼 수 있도록 했다.

  * 회원탈퇴 - 탈퇴 사유에 대한 내용을 select box로 제공해 빠르게 선택할 수 있도록 제공했다.

2. 장바구니
  -> 비회원 장바구니를 쿠키로 관리하고 있기 때문에 한페이지에 비회원과 회원에 대한 장바구니의
       구분을 넣었다. 상품의 수량에 대해 비동기로 수량을 늘리거나 줄여주고, 최대 10개 제한을 둔 후
       제고가 10개 미만인 경우 제고수를 넘을 수 없게 막아놓았다.
       각 상품마다 바로 구매,삭제 할 수 있는 버튼을 제공했고, 여러개일 경우 체크박스를 선택후 
       구매,삭제할 수 있도록 버튼을 제공했다. 

3. 비회원 주문조회
 -> 회원 주문조회와 비슷함(특징x)

4. 공지사항
 -> 페이징 처리를 하였고, 관리자만 글을 등록,수정,삭제 할 수 있게 버튼을 제공하였다.
     조회수 중복방지를 위해 쿠키로 관리해 회원일 경우에만 글상세보기로 갔을때 조회수가 증가하고,
      24시간이 지나야 다시 조회수가 올라간다.
